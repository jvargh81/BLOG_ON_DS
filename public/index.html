Scatter Plots

Data visualization using ggplot and ggiraph.

Using ggplot, we can display the data using graphs. This helps us to visualise the data.

  • Let’s inherit some libraries to start with
library(dplyr)
library(Lahman) 
library(devtools)
library(blogdown)
library(ggplot2)
library(ggiraph)
  • Adding a table for players with career Home run more than 400
df<-Batting%>%
  group_by(playerID)%>%
  summarize(career_HR=sum(HR),career_SO=sum(SO))%>%
  filter(career_HR >= 400)
  • We will now add first and last name of players instead of playerID
HR_vs_SO<-inner_join(df,Master,by=c("playerID"))%>%
  select(nameFirst,nameLast,career_HR,career_SO)
  • Here we have first and last name, we can use paste command to join both last name and first name
HR_vs_SO$name<-paste(HR_vs_SO$nameFirst,HR_vs_SO$nameLast)
  • Now using ggplot we can plot the graph for the table we generated
g<-ggplot()+
  geom_point_interactive(data=HR_vs_SO,aes(x=career_SO,y=career_HR,tooltip=nameLast,data_id=nameLast))+
  ggtitle("career home runs vs strike outs for great hitlers.")+
  xlab("career strikeouts")+
  ylab("career Homeruns")
  
  ggiraph(code=print(g),hover_css="Fill:white;stroke:blue")

Inserting libraries to use for the plots

  • Lets create a table for creating a bar graph.

Note : Bar graph are also single value graphs, but we can use x axis to display name or etc of that kind. text

df<-Teams%>%
  #filtering all the data for 1980 
  filter(yearID == 1980)%>%
  select(name,HR)%>%
  #Arranging all data in asscending order w.r.t HR, but we have to factor the level
  #inorder to display on the bargraph.
  arrange(HR)
  • We are just trying to plot the graph after using arrange api.Before factoring
ggplot()+
  # (dont do grouping or tallying))stat = identity means ggplot will do tallying for us.
  geom_bar(data=df,aes(x=name,y=HR),color="white",fill="purple",stat = "identity")+
  # Since the names are big and it will not fit all together, we can flip it over
  #using the below api.
  coord_flip()

Note : The above graph is not arranged, though using arrange api. We have to factor it inorder to arrange the data. text

  • Converting to factors or categorising it.*
# In this way we get arranged graph by leveling it w.r.t to the HR.
df$name<-factor(df$name,levels=df$name)
  • Now lets plot the graph using ggplot after arranging. After Factoring.
ggplot()+
  # (dont do grouping or tallying))stat = identity means ggplot will do tallying for us.
  geom_bar(data=df,aes(x=name,y=HR),color="white",fill="purple",stat = "identity")+
  # Since the names are big and it will not fit all together, we can flip it over
  #using the below api.
  coord_flip()+

  ggtitle("1980 Team Homeruns")+
  ylab("Homeruns")+
  xlab("Team")

Data visualization using ggplot and ggiraph.

df<-Master%>%
  # this helps to eliminate a empty entry.Here we are filtering only the one with value of weight.

  select(weight)

Introduction

dplyr is a new package which provides a set of tools to efficiently manipulate datasets in R. dplyr is the next iteration of plyr, focussing on only data frames. dplyr is faster and has a more consistent API and is easier to use.

With the help of these shown libraries, we will analyze and understand some of

the features of dplyr.

Note : Lahman is a baseball dataset. It contains information of baseball players.

library(Lahman)
library(dplyr)
